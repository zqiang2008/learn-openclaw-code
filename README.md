# Learn OpenClaw — 循序渐进掌握 AI 网关架构

## 项目定位

本项目是 [OpenClaw](https://github.com/openclaw/openclaw) 的**架构学习课程**

OpenClaw 是一个 92 万行 TypeScript 的**多渠道 AI 个人助理网关**。本课程用 12 个 Python 模块，从零构建一个简化版网关，让你从底层理解它的每一个设计决策。

## 核心思想

> **网关不是框架，网关是控制面。** 模型是智能的来源，网关只是把模型的能力连接到真实世界的管道。

OpenClaw 的架构可以用一句话概括：

```
用户 → 渠道(WhatsApp/Telegram/Discord/...) → 网关 → 路由 → Agent → LLM Provider → 响应 → 渠道 → 用户
```

本课程按照这个数据流，从左到右、从外到内，逐步构建每一层。

## 学习路径

### Phase 1: 基础架构层 — 搭建管道

| 模块 | 主题 | OpenClaw 对应 | 关键洞察 |
|------|------|--------------|---------|
| **s01** | Gateway 服务器 | `src/gateway/server/` | 一个 HTTP 循环就是网关的心跳 |
| **s02** | 消息路由 | `src/routing/` | 路由表决定消息去哪里 |
| **s03** | 渠道适配器 | `src/channels/` | 适配器模式统一不同平台 |
| **s04** | Provider 抽象 | `src/providers/` | 策略模式让 LLM 可插拔 |

### Phase 2: 智能响应层 — 注入智能

| 模块 | 主题 | OpenClaw 对应 | 关键洞察 |
|------|------|--------------|---------|
| **s05** | Agent 系统 | `src/agents/` | Agent 是有生命周期的状态机 |
| **s06** | 自动回复管道 | `src/auto-reply/` | 管道模式：接收→排队→处理→发送 |
| **s07** | 会话管理 | `src/sessions/`, `src/memory/` | 会话是上下文的容器 |
| **s08** | 插件架构 | `src/plugins/`, `src/plugin-sdk/` | 插件是网关的扩展点 |

### Phase 3: 高级机制层 — 完善控制面

| 模块 | 主题 | OpenClaw 对应 | 关键洞察 |
|------|------|--------------|---------|
| **s09** | CLI 命令系统 | `src/cli/`, `src/commands/` | CLI 是网关的操作界面 |
| **s10** | 配置系统 | `src/config/` | 分层配置 = 灵活 + 可控 |
| **s11** | 上下文与记忆 | `src/context-engine/`, `src/memory/` | 智能 = 模型 + 记忆 |
| **s12** | 完整网关 | 全部模块 | 所有机制的集成 |

## 快速开始

```bash
# 1. 克隆项目
cd learn-openclaw

# 2. 安装依赖
pip install -r requirements.txt

# 3. 配置环境
cp .env.example .env
# 编辑 .env，填入你的 API Key

# 4. 从第一课开始
python agents/s01_gateway_server.py
```

## 每课结构

每个学习模块由两部分组成：

1. **`agents/sXX_*.py`** — 可运行的 Python 实现，包含完整注释和 ASCII 架构图
2. **`docs/zh/sXX-*.md`** — 配套教学文档，含问题→方案→实现→对比的完整教学流程
